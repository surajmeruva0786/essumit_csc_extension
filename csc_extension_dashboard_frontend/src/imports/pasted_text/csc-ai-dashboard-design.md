Design a professional analytics dashboard UI for a government AI system called **“CSC AI Co-Pilot Dashboard”** used by government officials to monitor application submissions made through a Chrome Extension used by CSC operators.

The dashboard should follow a **clean Indian government portal style** similar to DigiLocker, eDistrict, or UMANG portals. Use a **formal color scheme** such as deep blue, white, light gray, and saffron/green accents inspired by Indian government portals. The interface should feel trustworthy, official, and minimal.

### Layout Structure

**Top Navigation Bar**

* Government emblem placeholder on the left
* Title: "CSC AI Co-Pilot Monitoring Dashboard"
* Search bar in the center
* Right side icons: notifications, admin profile avatar, logout

**Left Sidebar Navigation**
Use a vertical sidebar with icons and labels:

* Dashboard Overview
* Application Analytics
* Rejection Insights
* AI Model Performance
* CSC Operator Activity
* Citizen Notifications
* Service Categories
* System Logs
* Settings

### Main Dashboard Overview Page

At the top show **four large statistic cards**:

1. **Total Applications Submitted**

   * Large number counter
   * small subtitle "today / this week"
   * icon: document stack

2. **Acceptance Rate**

   * percentage
   * green upward trend indicator
   * subtitle "applications approved"

3. **Predicted Rejection Warnings**

   * number of warnings generated by AI
   * orange alert icon

4. **Active CSC Operators**

   * number of operators using extension today
   * user icon

### Analytics Charts Section

Below the cards add charts:

**1. Applications Over Time**

* line chart
* daily submission trends

**2. Acceptance vs Rejection Rate**

* stacked bar chart
* categories: Approved, Rejected, Pending

**3. Top Services Requested**

* horizontal bar chart
* examples:

  * Birth Certificate
  * Caste Certificate
  * Residence Certificate
  * Widow Pension
  * Old Age Pension

**4. AI Error Detection Insights**

* pie chart showing:

  * Document mismatch
  * Eligibility violation
  * Missing documents
  * Incorrect field values

### Operator Activity Table

Add a data table showing:

Columns:

* CSC Operator ID

Include sorting and filtering options.

### Rejection Insights Panel

A section showing the **top reasons for predicted rejection**:

Example cards:

* Aadhaar mismatch

Design should look like a **modern government digital platform**:

* clean grid layout
* rounded cards
* subtle shadows
* blue primary buttons
* official typography (Inter / Roboto)
* icons for documents, analytics, alerts, users

Sections:

The dashboard should feel like a **secure government monitoring platform used by administrators**, combining the aesthetics of Indian e-governance portals with modern analytics dashboards like PowerBI or Datadog.
